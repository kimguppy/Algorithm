1).해당 문제 해결 기법을 사용한 근거 (논리적인 근거)
① 문제인식
첫째 줄에 N, M
다음 N개의 줄에는 N개의 정수
다음 M개의 줄에는 a, b의 쌍

② 문제 유형 파악:
N(1 ≤ N ≤ 100,000)개의 정수들이 있다.
a, b의 쌍이 M(1 ≤ M ≤ 100,000)개 주어졌을 때,
a번째 정수부터 b번째 정수까지 중에서 제일 작은 정수를 찾는 것

=> 세그먼트 트리 사용

2). 의사 코드 또는 DFA 다이어그램 작성 (대략적으로 작성)
➀ SegmentTree N개의 정수 잎으로 하는 트리 생성
➁ isPossible:  압축이 가능한지 해당 공간을 체크해주는 함수
➂ QuadTree(int x, int y, int size): 압축이 가능할 경우 하나의 색상으로 압축해준다.
압축이 불가능 할 경우 사이즈를 절반으로 나누어야 한다.
-> QuadTree 4분할 해서 다시 재귀
